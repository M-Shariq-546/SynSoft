EmailJS templates (reference for dashboard)

Copy each block into EmailJS → Email Templates → Content (HTML). Use inline styles only. Variables are double-brace as EmailJS expects.

From index.html you send: user_name, first_name, last_name, user_email, company, service, service_value, service_badge, message

  • Template notify: To = fixed synsoftco@gmail.com — Subject e.g. New lead: {{service_badge}}
  • Template confirm: To = {{user_email}} (EmailJS: use “To” field in template) — Subject e.g. We received your message, {{first_name}}. If this “To” is wrong or the variable name does not match index.html (user_email), the second /send call returns 422 while the notify email to Synsoft can still return 200.

1) Internally — new inquiry (Synsoft)

2) Visitor — professional confirmation (light card layout)

Allowlist your production origin in the EmailJS account, then set EMAILJS_* in index.html to the public key, service id, and both template ids.

The confirmation template (section 2) uses the same fields as buildParams() in index.html (first_name, user_name, user_email, company, service, service_badge, message). For a time field, add it in that object and a matching row in the HTML.
